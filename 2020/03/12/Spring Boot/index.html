Spring Boot

简介

  • 简化Spring应用程序的创建和开发过程
  • 抛弃繁琐的xml配置过程，采用注解配置
  • 快速创建基于Spring框架的应用程序，让编码、配置、部署、监控变简单
  • 内嵌Tomcat、Jetty服务器，不需要部署war包文件
  • 提供约定的starter POM来简化Maven配置，让Maven的配置变得简单
  • 自动配置Spring、Spring MVC等
  • 提供了程序的健康检查等功能
  • 内部依然使用SSM等框架
  • 一个快速开发的脚手架
  • 基于SpringBoot可以快速的开发单个微服务

四大核心

  • 自动配置✔
  • 起步依赖✔
  • Actoator（健康检查）
  • 命令行界面

第一个Spring Boot程序

  • 基本POM文件
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <!-- 父级依赖 -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.8.RELEASE</version>
        <relativePath/> <!-- lookup parent from repository -->
    </parent>

    <groupId>com.lyk</groupId>
    <artifactId>hello-spring-boot</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <name>hello-spring-boot</name>
    <description>Demo project for Spring Boot</description>

    <!-- 属性配置 -->
    <properties>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
        <java.version>1.8</java.version>
    </properties>

    <dependencies>

        <!-- Spring Boot 开发Web项目的起步依赖 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- 测试起步依赖 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>

            <!-- SpringBoot提供的项目编译打包插件 -->
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>
  • Spring Boot入口main方法
@SpringBootApplication //该注解是SpringBoot项目的核心注解，主要是开启Spring自动配置
public class HelloSpringBootApplication {

    public static void main(String[] args) {
        // 启动了SpringBoot程序，启动Spring容器，启动内嵌的Tomcat
        SpringApplication.run(HelloSpringBootApplication.class, args);
    }

}
  • 创建Spring MVC 的Controller类
@Controller
public class HelloController {
    @ResponseBody
    @RequestMapping("/boot/hello")
    public String hello() {
        return "hello Spring Boot."
    }
}

Spring Boot 使用拦截器

  • 编写一个拦截器类

    • 实现HandlerInterceptor接口

  • 编写一个配置类

    • 继承WebMvcConfigurerAdapter类
    • 为该配置类添加@Configuration注解，让Spring Boot扫描类
    @Configuration
public class WebConfig extends WebMvcConfigurerAdapter

    // 需要拦截的路径
    String[] addPathPatterns = {
        "",
        ""
    };

    // 不拦截的路径
    String[] excludePathPatterns = {
        "",
        ""
    };
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new LoginInterceptor()).addPathPatterns().excludePathPatterns();
    }

Spring Boot 集成 Thymeleaf模板

  • 认识Thymeleaf：Thymeleaf模板基于HTML，在非web环境下直接显示模板上的静态数据；在web环境下能像JSP一样从后台接收数据并替换掉模板上的静态数据

  • 步骤

    1. 引入Thymeleaf依赖、

    2. 在Spring boot的核心配置文件对Thymeleaf进行配置

      # 开发阶段，建议关闭thymeleaf的缓存
spring.thymeleaf.cache = false
# 使用遗留的html5以去掉对html标签的校验
spring.thymeleaf.mode = LEGACYHTML5

      如果不想对标签进行严格的验证，使用spring.thymeleaf.mode = LEGACYHTML5去掉验证，去掉该验证，则需要引入如下依赖，否则报错

      nekohtml
unbescape

    3. 写一个Controller去映射到模板页面

      @RequestMapping("/index")
public String index(Model model) {
    model.addAttribute("data", "恭喜，Spring boot集成thymeleaf成功！")
        return"index";
}

    4. 在/resources/thymeleaf下写一个index.html文件

      <!-- 需要在html标签中加入属性：xmlns:th="http://www.thymeleaf.org" -->
<html xmlns:th="http://www.thymeleaf.org">
    <head>

    </head>
    <body>
        <p th:text="${data}">hello world </p>
    </body>
</html>

      静态资源放置在/resources/static目录下
